## 适合接入哪些工具

QLHazyCoder API 这类中转站，最常见的接入方式是把它当作模型 API 提供方使用。具体能否接入，取决于后台提供的接口格式和你的客户端是否支持自定义 API 地址。

常见可测试场景包括：

- Claude Code、Cline：重点测试 Claude/Anthropic 兼容接口和模型名
- Cursor、ChatBox：重点测试 OpenAI 兼容接口、Base URL 和 API Key
- OpenAI SDK 项目：重点测试 `/v1/chat/completions` 或平台实际支持的接口
- 自写脚本：重点测试模型列表、流式输出、错误码和账单消耗
- 图片生成工作流：重点测试 `image2` 的文生图、图生图和下载体验

不要凭文章里的模型名直接写死配置。最稳妥的做法是从后台复制当前可用模型名，再填到客户端里。
