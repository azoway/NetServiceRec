# 🧩 一、什么是 IX 线路？

**IX（Internet Exchange，互联网交换中心）线路**  
是运营商在 IX 交换中心直接互联的高速网络，路由更短、延迟更低。

对比传统公网：

| 传统公网       | IX 线路  |
| ---------- | ------ |
| 多跳、绕路、拥塞   | 更短路由   |
| 高峰期延迟、丢包明显 | 稳定高速   |
| 公网互联质量不可控  | 节点质量稳定 |

### ⭐ IX 线路优势

*   **低延迟**：适合游戏、语音、远程办公
*   **高并发大带宽**
*   **跨境访问稳定**
*   **成本远低于专线**

# 🛠 五、搭建思路（流程级说明）

> ⚠️ 以下仅介绍网络架构与配置流程，不包含任何违法用途。请务必遵守当地法律法规 & 服务商 ToS。

***

## ① VPS 部署 SS 服务端

SSH 登录 VPS 后执行：

```bash
bash <(curl -s https://raw.githubusercontent.com/azoway/across/main/gost/gost.sh)
```

记录：

*   端口
*   密码
*   加密方式

***

## ② AKCDN 创建转发规则

路径：

    控制面板 → 规则管理 → 新建转发

填写：

*   前端端口：你希望前置访问的端口
*   后端目标：你的 VPS IP + SS 端口
*   协议：TCP

保存 → 等待生效。

***

## ③ 阿里云 G 口配置 NAT 转发

开放端口 → nftables 转发到 AKCDN

使用脚本（来自 GitHub）：

```bash
bash <(curl -s https://raw.githubusercontent.com/azoway/across/refs/heads/main/nftables/nft-nat.sh)
```

确保：

*   nftables 加载成功
*   端口正确映射到 AKCDN

# 🖥 六、如何在本地使用？

准备：

    阿里云前置 IP + 端口

在本地 SS 客户端：

*   地址：阿里云前置 IP
*   端口：阿里云转发端口
*   密码：VPS SS 的密码
*   加密方式：与 VPS 一致

路径如下：

    你 → 阿里云 G 口 → AKCDN IX → VPS（落地）

效果：

*   延迟稳定
*   高峰不卡
*   大流量传输更顺畅
