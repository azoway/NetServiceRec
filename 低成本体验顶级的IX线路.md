## 一、什么是 IX 线路？

**IX（Internet Exchange，互联网交换中心）线路**，指的是利用各运营商之间在 IX 交换点建立的**高速直连网络**，实现不同网络之间更低延迟、更大带宽的一种线路形态。

简单理解：

*   传统公网：
    *   数据在不同运营商 / 地区之间绕来绕去
    *   路由跳数多，延迟高，丢包也更容易发生
*   使用 IX 线路：
    *   运营商在 IX 交换中心直接互联
    *   网络路径更短、更干净，延迟和抖动明显降低

### IX 线路的典型优势

*   **延迟更低**：  
    对游戏、语音、实时应用非常友好，跨境访问也更稳定。
*   **带宽更大、更稳定**：  
    大流量下载、音视频传输不容易“卡脖子”。
*   **路由更可控**：  
    通过 IX 中转，可以规避一部分质量差、绕路严重的公网路径。
*   **性价比高**：  
    相比传统专线便宜非常多，但在体验上能接近专线。

### 适合哪些场景？

*   对网络质量敏感的远程办公 / 研发
*   游戏加速、低延迟实时应用
*   跨境业务访问国内 / 国外服务器
*   大流量传输、内容分发中转（如 CDN 回源优化）

## 二、整体架构：如何用低成本拼一条 IX 线路？

本文介绍的低成本玩法，大致架构是：

1.  **落地 VPS（亚太地区）**
    *   部署 SS 等代理服务，作为真正的“落地点”。
2.  **AKCDN IX 线路作为中转层**
    *   利用 AKCDN 的 IX 节点，把流量从前置拉到落地 VPS。
3.  **前置节点：阿里云 G 口（广州）**
    *   通过闲鱼等渠道购买带宽充足的 G 口机器
    *   用户本地只需要连接阿里云前置 IP + 端口即可。

> 这样一套组合拳下来，整体效果类似“前置节点（本地就近） → IX 中转 → 亚太落地 VPS”，既兼顾了**速度**，又把**成本**压到比较低的水平。

## 五、IX 线路搭建思路（高层流程说明）

> ⚠️ 提示：下面只做 **原理级和流程级说明**，不提供可直接执行的网络穿透 / 代理绕行脚本指令。  
> 在实际操作时，请务必遵守所在地区法律法规及服务商使用条款，只在合法、合规的前提下使用网络加速与中转服务。

整个搭建过程可以拆成三步：

### 步骤 1：在亚太落地 VPS 上部署 SS 服务端
目标：在 VPS 上部署一个基于 **SS 协议** 的服务端，用于承接来自 AKCDN 的流量。
1.  使用 SSH 登录 VPS
2.  更新系统、安装必要依赖
3.  根据你选择的实现方式（如 xray、gost 等），参考其官方文档：
    *   下载并安装程序
    *   生成配置文件（端口、加密方式、认证信息）
    *   设置开机自启与日志

> SS搭建的一键脚本：
```bash
bash <(curl -s https://raw.githubusercontent.com/azoway/across/main/gost/gost.sh)
```

### 步骤 2：在 AKCDN 上配置转发规则（指向落地 VPS）

目标：让 AKCDN 的 IX 节点，把流量转发至刚刚部署好的 VPS SS 服务端。

一般会包含以下思路：

1.  登录 AKCDN 控制台
2.  创建一个新的转发 / 规则：
    *   前端监听端口、协议
    *   后端指向你的 VPS IP + 端口
3.  确认：
    *   规则数没有超过套餐限制
    *   带宽、流量配置适合你的使用强度
4.  保存并应用配置，等待规则生效

### 步骤 3：在阿里云 G 口前置节点上做转发

目标：用户本地 → 阿里云 G 口 → AKCDN → VPS

在阿里云 G 口机器上，一般会：

1.  开放必要端口（安全组 + 防火墙）
2.  配置 NAT / 端口转发，将本地访问的端口转发到 AKCDN 节点
3.  确认系统防火墙规则（如 nftables）生效且没有误拦截

> 同样地，关于具体命令、脚本或 nftables 配置，建议严格按照服务商或项目的官方文档进行，避免因配置错误造成安全问题或被判定为违规使用。
> 一键脚本：
  
```bash
https://raw.githubusercontent.com/azoway/across/refs/heads/main/nftables/nft-nat.sh
```

## 六、本地如何使用这条 IX 线路？

当上面三层架构都配置好、规则生效后：

1.  你会有一个 **阿里云前置节点的 IP + 端口**
2.  在本地客户端里（如支持 SS 协议的工具）：
    *   填写阿里云前置 IP
    *   端口设为你在前置节点上设置的监听端口
    *   其他信息与 VPS（SS 服务端配置）保持匹配

此时，数据路径将会是：

> **本地客户端 → 阿里云 G 口（广州） → AKCDN IX 节点 → 亚太 VPS 落地**

在网络体验上，通常能明显感知到：

*   延迟更稳定
*   丢包更少
*   高峰期可用带宽更有保障

## 八、注意事项与风险提示（建议放在文末，提升可信度）

1.  **法律与合规性**
    *   不同地区对网络加速、代理、跨境访问有不同规定
    *   在搭建和使用 IX 线路前，请了解并遵守相关法律法规
    *   严禁用于任何违法或侵权行为

2.  **服务条款（ToS）**
    *   阿里云、AKCDN、VPS 服务商都有各自的使用条款
    *   滥用带宽、攻击他人、跑灰色业务都有可能被封号
    *   建议合理控制流量与并发，避免触碰红线

3.  **安全问题**
    *   不要随便执行来源不明的一键脚本
    *   强密码、及时更新系统，防止服务器被入侵
    *   对重要服务开启日志与告警，发现异常及时处理
